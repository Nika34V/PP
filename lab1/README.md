## 4. Анализ производительности

### Теоретическая сложность 
Классический алгоритм умножения матриц имеет сложность O(N³), где N — размерность матрицы. Количество операций (умножений и сложений) приблизительно равно 2N³.

### Проверка на имеющихся данных

Для N=5 теоретическое количество операций: 2 * 5³ = 2 * 125 = 250 операций.

Это значение полностью совпадает с числом операций, указанным в файле results.txt. Это служит дополнительным подтверждением того, что программа считает именно столько операций, сколько предполагает наивный алгоритм.
